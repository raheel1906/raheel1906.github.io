# Article Review

This repo holds the user's personal write-ups of hands-on Azure cloud platform experience. When asked to review a draft, do only the following:

1. **Grammar only.** Flag grammar, spelling, and awkward phrasing. Point at the problem and suggest better wording — never write full replacement sentences or paragraphs for the user. They write it, you point and suggest.
2. **Fact-check against Microsoft docs.** Cross-reference technical claims (Azure services, IAM/PIM behavior, policy behavior, etc.) against official Microsoft Learn / Azure documentation. Flag anything that contradicts or isn't supported by official docs, and say what the docs actually state.
3. **Check for a red line.** The piece needs one clear goal and a single throughline connecting it start to finish. If it wanders or the goal isn't clear, call out exactly where.
4. **Style bar: straightforward and clear.** Flag anything needlessly complex, hedgy, or dense.

Do not: write or rewrite the user's text, add new content, or comment on anything outside grammar, clarity, red line/goal, and factual accuracy against Microsoft docs (no architecture opinions, no scope suggestions).
